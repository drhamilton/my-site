# my-site

Personal portfolio site used in interviews — to show who Dillon is and what he's
building. The centre of gravity is the **Projects** area.

## Language

**Project**:
A single piece of work shown in the portfolio. Always appears as a Card; may or
may not have a Walkthrough. Some Projects only link out to a Demo or repo.
_Avoid_: portfolio item, entry, work

**Card**:
The compact summary of a Project shown in the Projects grid (thumbnail, title,
blurb, tech tags, links). Every Project has exactly one Card. A Card may link
through to a Walkthrough, out to a Demo/repo, or both.
_Avoid_: tile, thumbnail (the thumbnail is just one part of a Card)

**Walkthrough**:
The dedicated, longer-form page for a Project — narrative (problem / approach /
result) plus screenshots. Optional: a Project can be Card-only.
_Avoid_: case study, write-up, detail page, post

**Demo**:
A running, interactive version of a Project that a visitor can use, linked from
the Card/Walkthrough. Distinct from the source repo link.
_Avoid_: live site, example, sandbox
